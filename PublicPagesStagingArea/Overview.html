XQuery and XPath Full Text 1.0 Test Suite

Overview

The XQuery and XPath Full Text 1.0 Test Suite (XQFTTS) provides a set of metrics for determining whether XPath and XQuery Full Text 1.0 can be implemented interoperably as published. XQFTTS will help implementers identify possible problems both with the Specification and with their software. The test suite is occasionally updated to reflect the current work of the W3C XML Query Working Group and the W3C XSL Working Group. All releases of the XQFTTS will be available from this page.

The XQFTTS contains over 400 test cases. It contains a catalog that contains general information on the test suite, as well as test descriptions for each of the test cases included in this release. Test queries and expected results are contained in individual files. All interpretations of this Candidate Recommendation are subject to confirmation by the XML Query Working Group and the XSL Working Group.

Implementors are encouraged 1) to write a harness for this test suite and to test their implementations, 2) to report their results to the XML Query and XSL Working Groups, and 3) to submit test cases to the XML Query and XSL Working Groups to help us complete our coverage of XPath and XQuery Full Text 1.0. All of these areas are discussed in greater detail below.

This document serves as a first point of information for all interested parties. A general account of current activities, finalized work, and pointers to further information is given. In the sections below, you will be able to get a good view of the status of the XQFTTS, and if you are interested in something in particular you are invited to follow the appropriate link given below.

Status

The XPath and XQuery Full Text 1.0 specification has been published as a Candidate Recommentation on 2008-05-16.

XQFTTS 0.8.0 provides an early look at this test suite. It contains about 90% of the tests that will exist when the test suite has been completed (except for tests expressed in XQuery's XML syntax, XQueryX). We encourage implementors to build their test harnesses and to provide feedback to the XML Query and XSL Working Groups.

XQFTTS 0.8.0 contains no Full Text XQueryX test cases. We expect to have a Full Text XQueryX test case that corresponds to each Full Text test case that is not identified as a parse error when the XQFTTS is complete.

It will take us some time to complete this test suite. We encourage members of the public to contribute tests in some specific areas. The submission of test cases is discussed below.

Running the Test Suite

We encourage you, the implementors of XPath and XQuery Full Text 1.0, to run these tests and send us your results. These results will help us exit Candidate Recommendation (described in the W3C Process Document).

As there is no de jure or de facto API for implementations of XQuery 1.0, we are not able to provide a test harness to execute these tests. You will have to write your own test harnesses.

We have provided Guidelines for Running the XPath and XQuery Full Text 1.0 Test Suite to assist you in running these tests. These guidelines will tell you how the queries may be modified before they are executed and how to compare your actual results with the expected results that we have provided.

Reporting a Problem

If you believe that some of the queries or their expected results are incorrect, then we encourage you to enter bug reports in the public instance of Bugzilla at W3C, http://www.w3.org/Bugs/Public/. To enter a bug report you will need a Bugzilla signon (email address and password). You can create this signon by following the Open a new Bugzilla account link on the Bugzilla Main Page.

General information on Bugzilla, an open-source bug tracking system, is available at http://www.bugzilla.org/. The standard documentation for the version currently being run on the W3C server is available at http://www.bugzilla.org/docs/2.18/html/index.html.

To enter a bug report in Bugzilla you will need to log in, choose the action "New", and then choose the "XPath and XQuery Full Text 1.0 Test Suite" product. Enter a short description of the problem in the Summary field; it would be very helpful if every Summary started with the string "[XQFTTS]". Detailed information should be entered in the Description field. Please be specific about the problem that you are reporting and clearly identify the test case or test cases that are problematic. If you believe that the expected result is wrong, then tell us what result you believe is correct. If you believe that another result for a test case is justified, then please tell us what it is. In both cases, please cite portions of one or more of the XPath, XQuery, and Full Text specifications that support your position.

Submission of Test Suite Results

When you are ready to submit your results to us, we ask that you follow the guidelines provided in Guidelines for Submitting XPath and XQuery Full Text Test Suite Results. These guidelines will tell you how to generate an XML result document that we can use to generate our reports. They will also tell you how to generate your own report from the XML result file.

If you have indicated that you wish your results to be displayed as “Anonymous”, then please send them to the XML Query WG Team Contact, Liam Quin, cc'ing w3t-archive@w3.org. Otherwise, please send them to the Full Text Task Force chair, Jim Melton , cc'ing www-archive@w3.org.

We encourage you to send us results early, and then to send us updates as the FTTS progresses and as your implementation progresses.

Submission of Test Cases

We welcome submissions of test cases from individuals and organizations that are not already participating in the XML Query Working Group or the XSL Working Group. We encourage you to look at Guidelines for XPath and XQuery Full Text Test Suite Submissions so that the test cases that you submit can be added easily to our test suite.

If you would like to submit test cases, then you must:

  • Register for a W3C account. You can request one through the Public Access Request Form.
  • Submit your test cases using Grant II: Grant of License for Contributed Test Cases Published Outside a W3C Recommendation. This license grant, for non-Participant test case Contributors, is described in Policies for Contribution of Test Cases to W3C.

The XML Query Working Group and XSL Working Group will, at their sole discretion, determine what tests are incorporated in the test suite.
